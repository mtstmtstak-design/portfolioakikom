医療サイトのUI/UX改善とUD対応

ー 利用目的に素早くたどり着ける、ストレスの少ない医療サイト設計 ー

サイト　2025.7-8

地方病院の既存Webサイトを題材に、情報の探しにくさや操作性の課題を洗い出しユニバーサルデザインの観点からUI改善を行った自主制作。
幅広い年齢層が迷わず利用できる医療サイトを目指した。

●制作範囲
　情報整理 / 画面デザイン　　　2週間
　コーディング　　　　　　　　１週間
　改善ポイントの整理・比較資料作成　3日間

●制作ツール
　Figma / Visual Studio Code

使用色を再検討
前回同様青をメインカラーとし、イエロー系をアクセントにすることで 視認性UP

色覚多様性に配慮
画像はP型（一型）視覚の見え方である。問題なく色差が確認可能

大きめのボタンで押しやすく
ボタンの大きさを十分に設け、押しづらさを解消

詳細ページの作成
TOPページのほか、「病院について」のページも作成
アクセントカラーを取り入れることで単調になりすぎない構成とした

目的
  • 必要な情報に短時間でたどり着ける医療サイト設計
  • 高齢者・視覚特性に配慮し、操作ストレスを軽減
  • 情報の重要度を整理し、直感的に理解できる構成にする
ターゲット
  • 当該病院の利用者およびその家族
  • 高齢者を含む、Web操作に不慣れなユーザー
課題
  • デザインの統一感がなく、視線誘導が弱い
  • 操作可能な要素が分かりにくい
  • ボタンサイズが小さく、操作しづらい箇所が多い
デザイン・情報設計
  • メインカラー＋アクセントカラーで情報の強弱を明確化
  • 文字サイズは最小16pxとし、可読性を確保
  • 利用頻度の高い情報を上部に配置
  • 高さを取る情報はアコーディオンで整理
資料作成 改善内容については、Before / After を比較した資料として整理。
UI・アクセシビリティ・UXの3観点から、設計意図と改善ポイントを可視化した。
